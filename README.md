## Overview

RetiFlux™ is a highly adaptable, web-based production documentation and inventory management system designed for warehouses and manufacturing facilities of any size handling any product type. From small distribution centers to enterprise-scale operations—RetiFlux™ scales seamlessly to your needs.

## Etymology

- **Reti** - Latin *retis* (net, network, grid, web)
- **Flux** - Latin *fluxus* (flow, continuous movement)
- **Meaning:** "Network Flow" — the flowing network/grid that connects all warehouse operations

## Quick Start

### Requirements
- Web browser (Chrome, Edge, Safari, Firefox)
- Google account with Sheets access
- GPS-enabled device for geofenced access (optional)
- Network connectivity (initial setup only)

### Setup Time
**15-30 minutes** for basic configuration

### Files

- **`bandingtickets.html`** - Main application
- **`google-apps-script-code.js`** - Google Sheets backend
- **`config.example.js`** - Configuration template
- **`RetiFlux_Product_Brief_FINAL.html`** - Product documentation

---

## Implementation

1. Create Google Sheet from provided template
2. Deploy `google-apps-script-code.js` as web app
3. Copy `config.example.js` to `config.js` and configure credentials
4. Set QR code base URL
5. Add authorized users to database
6. Map SKUs to warehouse zones
7. Deploy `bandingtickets.html` to web server or GitHub Pages
